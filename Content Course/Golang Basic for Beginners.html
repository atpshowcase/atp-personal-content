CHAPTER A. Introduction and Setup

A.1 What is Go and Why It Matters

Go is a programming language designed to be simple, fast, and easy to maintain. Its philosophy is clear: don't make syntax complicated, keep performance fast, and make concurrency first-class citizens.

In the working world, Go is often chosen for backend services, microservices, CLI tools, and cloud systems because its builds are fast and deployment is easy.

package main

import "fmt"

func main() {
    fmt.Println("Hello Go")
}
Analogy: Go is like a daily motorcycle that's simple, fuel-efficient, but still strong for long journeys.

A.2 History and Philosophy of Go

Go was born at Google to solve large-scale software problems: long compile times, complicated dependencies, and heavy codebase maintenance.

  • Simplicity: coding style is made consistent.
  • Performance: compiled language, native binary.
  • Concurrency: goroutines and channels built-in.

Unique facts that make Go different:

  • Go started being developed at Google around 2007 by Robert Griesemer, Rob Pike, and Ken Thompson.
  • Go was open-sourced to the public in 2009, then released stable Go 1 in 2012.
  • Go has a Go 1 compatibility promise, so old code generally still runs in new versions.
  • Tool gofmt became the official standard format, so Go team coding styles are usually consistent without long debates.
  • Go's famous mascot is the Gopher, its illustration was created by Renee French.
go version

go env
Analogy: compared to Swiss army knives that have everything, Go is more like a main kitchen knife that's focused, sharp, and used every day.

A.3 Installing Go and Environment Setup

Minimum setup: install Go, ensure PATH is correct, then test first program. For modern projects, use modules from the start so dependencies stay organized.

go version

mkdir belajar-go
cd belajar-go
go mod init belajar-go

go run main.go
Analogy: before cooking, you check if the stove is on, gas is safe, and ingredients are ready on the table.

CHAPTER B. Basic Syntax and Data

B.1 Program Structure in Go

Every executable Go program starts with package main and func main(). Import is used to get standard packages or your own packages.

package main

import "fmt"

func main() {
    fmt.Println("Program started")
}
Analogy: package main is the venue entrance door, func main is the event's starting point.

B.2 Variables and Basic Data Types

In Go, you can declare variables using var or shorthand :=. The most commonly used basic types: int, float64, bool, string.

When to use var vs :=: use := when you initialize directly within function. Use var when you need declaration without initial value, need explicit type, or declare at package level.

Reason why you don't have to over-declare types (like always writing string, int, float manually): Go compiler can already infer type from initial value. Code becomes more concise, stays type-safe, and is faster to read during review.

  • Less noise, so focus stays on logic, not type repetition.
  • Stays safe because Go is strongly-typed, not dynamically-typed.
  • Refactoring is easier because initial value changes automatically follow relevant type inference changes.
var umur int = 25
kota := "Bandung"
aktif := true
nilai := 87.5

fmt.Println(umur, kota, aktif, nilai)

Go also supports multiple variable declarations at once, making it convenient for related values.

var (
          firstName string = "Rani"
          lastName  string = "Saputra"
          isMember  bool   = true
      )

      lat, lng := -6.2, 106.8

      fmt.Println(firstName, lastName, isMember)
      fmt.Println("koordinat:", lat, lng)

Important note: Go doesn't allow variables to be declared but not used. This is intentional so code stays clean and minimizes dead code.

Analogy: variables are storage drawers, each drawer has a clear content label.

B.3 Zero Values and Constants

If variable is declared without initial value, Go automatically gives zero value. This makes default behavior more predictable.

  • int becomes 0
  • bool becomes false
  • string becomes ""
var total int
var isReady bool
var nama string

const appName = "BelajarGo"

fmt.Println(total, isReady, nama, appName)

Besides that, other types also have built-in zero values:

  • pointer becomes nil
  • slice becomes nil (len 0, data not allocated)
  • map becomes nil (must make before filling)
  • struct contains zero value from each of its fields
var p *int
      var tags []string
      var scores map[string]int

      type User struct {
          Name string
          Age  int
      }

      var u User

      fmt.Println(p == nil)         // true
      fmt.Println(tags == nil)      // true
      fmt.Println(scores == nil)    // true
      fmt.Println(u.Name, u.Age)    // "" 0

Constants suitable for values that shouldn't change like tax rate, maximum retry limit, or application name.

const (
          taxRate   = 0.11
          maxRetry  = 3
          appRegion = "ap-southeast-1"
      )

      fmt.Println(taxRate, maxRetry, appRegion)
Analogy: if you haven't filled the form, the system still gives default values so the flow doesn't break.

B.4 Type Conversion and Operators

Go is strict about data types, so conversions must be explicit. Standard operators are used for calculation, comparison, and making logic.

Why not use built-in decimal type? Go intentionally keeps primitive types small and simple. Because of that, there's no built-in decimal like in some other languages.

For money/accounting needs, safe practices are usually:

  • Save values as smallest integer units (example: full rupiah or cents) so precision stays stable.
  • If you need high non-binary precision, use packages like math/big or special decimal libraries.
a := 10
b := 3

hasil := a + b
isBesar := a > b
valid := (a > 0) && (b > 0)

rata := float64(a) / float64(b)

fmt.Println(hasil, isBesar, valid, rata)
// Contoh uang disimpan sebagai integer
harga := 150000   // Rp150.000
diskon := 12500   // Rp12.500
total := harga - diskon
fmt.Println(total) // 137500

Because of strong typing, int doesn't automatically convert to float64. You must convert manually so calculation results match expectations.

price := 15000
      discount := 12.5

      finalPrice := float64(price) - (float64(price) * discount / 100)
      fmt.Println(finalPrice)

Examples of operators commonly used in daily code:

  • Arithmetic: +, -, *, /, %
  • Comparison: ==, !=, >, <, >=, <=
  • Logical: &&, ||, !
  • Combination assignment: +=, -=, *=
x := 7
      y := 2

      fmt.Println(x % y)         // 1
      fmt.Println(x == y)        // false
      fmt.Println(x > 5 && y > 1) // true

      x += 3
      fmt.Println(x) // 10

Tips: be careful with integer division. 5 / 2 results in 2, not 2.5. If you want decimals, convert to float64 first.

Analogy: items of different sizes must be adjusted first before entering the same shelf.

CHAPTER C. Control Flow and Functions

C.1 If Else and Switch

Control flow is used to direct program paths based on conditions. Go also has short statements in if that are often used for quick validation.

nilai := 82

if nilai >= 85 {
    fmt.Println("Grade A")
} else if nilai >= 70 {
    fmt.Println("Grade B")
} else {
    fmt.Println("Need practice")
}

switch {
case nilai >= 90:
    fmt.Println("Excellent")
case nilai >= 75:
    fmt.Println("Good")
default:
    fmt.Println("Keep learning")
}
Analogy: like a parking attendant who directs vehicles to different lanes according to field conditions.

C.2 Looping with For

Go only has one loop keyword: for. But its form is flexible, can be classic style, similar to while, up to infinite loop.

for i := 1; i <= 3; i++ {
    fmt.Println("for klasik", i)
}

n := 0
for n < 2 {
    fmt.Println("for ala while", n)
    n++
}

for {
    fmt.Println("infinite loop")
    break
}
Analogy: one remote can do many functions, just different usage modes.

C.3 Functions and Multiple Return Values

Functions in Go can return more than one value. This is a core feature often used together with error handling.

func bagi(a int, b int) (int, int) {
    return a / b, a % b
}

hasil, sisa := bagi(10, 3)
fmt.Println(hasil, sisa)
Analogi: sekali transaksi bisa dapet struk dan kembalian sekaligus.

C.4 Variadic and Anonymous Functions

Variadic functions accept flexible number of arguments. Anonymous functions are suitable for quick operations or small callbacks.

Variadic uses ... before parameter type. Internally, this parameter is treated like a slice. So you can send individual values or directly pass a slice using ... too.

func jumlah(nums ...int) int {
    total := 0
    for _, n := range nums {
        total += n
    }
    return total
}

double := func(x int) int {
    return x * 2
}

fmt.Println(jumlah(1, 2, 3, 4))
fmt.Println(double(5))
angka := []int{10, 20, 30}
fmt.Println(jumlah(angka...)) // slice di-unpack jadi argumen variadic

What does underscore (_) mean in for range? In Go, range usually returns two values: index and value. The _ symbol means that value is intentionally ignored because it's not used.

for i, n := range nums {
  fmt.Println(i, n) // pakai index dan value
}

for _, n := range nums {
  fmt.Println(n) // index diabaikan
}

for i := range nums {
  fmt.Println(i) // cuma index
}

Anonymous function is also often used as closure, which is a function that "remembers" variables in outer scope.

counter := func() func() int {
  x := 0
  return func() int {
    x++
    return x
  }
}

next := counter()
fmt.Println(next())  // 1
fmt.Println(next())  // 2
fmt.Println(next())  // 3
fmt.Println(next())  // 4
fmt.Println(next())  // 5

next2 := counter()

fmt.Println(next2())  // 1
fmt.Println(next2())  // 2
fmt.Println(next2())  // 3
fmt.Println(next2())  // 4
fmt.Println(next2())  // 5

Maksud contoh closure di atas:

  1. counter() membuat variable x dengan nilai awal 0.
  2. counter() mengembalikan function dalam yang tetap punya akses ke x.
  3. Saat next := counter(), function next membawa "memori" miliknya sendiri.
  4. Panggilan pertama next() membuat x jadi 1, panggilan kedua jadi 2, dan seterusnya.
  5. Kalau bikin next2 := counter(), maka next2 punya x baru (tidak berbagi state dengan next).
next := counter()
next2 := counter()

fmt.Println(next())  // 1
fmt.Println(next())  // 2
fmt.Println(next2()) // 1 (state terpisah)
Analogi: closure itu kayak tiap function punya dompet pribadi. Isi dompetnya (state) tetap kebawa dan bertambah tiap kali function itu dipanggil.

BAB D. Arrays Slices Maps and Structs

D.1 Arrays and Slices Core Concept

Array punya ukuran tetap, slice lebih fleksibel dan dipakai hampir di semua project Go. Memahami slice itu wajib kalau mau nyaman nulis Go.

arr := [3]int{10, 20, 30}
sl := []int{1, 2, 3}

sl = append(sl, 4)

fmt.Println(arr)
fmt.Println(sl)
fmt.Println(len(sl), cap(sl))
Analogi: array itu rak permanen, slice itu keranjang belanja yang bisa nambah isi.

D.2 Slice Operations and Range Iteration

Lo bisa ambil bagian slice, iterasi pakai range, dan manfaatin len/cap buat baca kondisi data saat runtime.

data := []string{"go", "python", "java", "rust"}

subset := data[1:3]
fmt.Println(subset)

for idx, val := range data {
    fmt.Println(idx, val)
}
Analogi: kayak motong playlist jadi segmen favorit terus diputar ulang sesuai urutan.

D.3 Maps for Key Value Data

Map dipakai untuk data berbasis pasangan key-value. Untuk akses data berdasarkan key, map jauh lebih efisien dibanding cari manual di slice.

skor := map[string]int{
    "Ayu": 92,
    "Bima": 85,
}

skor["Citra"] = 88

if val, ok := skor["Ayu"]; ok {
    fmt.Println("Skor Ayu", val)
}
Analogi: map itu buku kontak, lo cari langsung berdasarkan nama, bukan scroll semua daftar dulu.

D.4 Structs and Methods Basics

Struct dipakai buat model data domain. Method pada struct bikin behavior nempel ke datanya, jadi kode lebih rapi.

type User struct {
    Name string
    Age  int
}

func (u User) Intro() string {
    return fmt.Sprintf("Halo, saya %s", u.Name)
}

u := User{Name: "Raka", Age: 24}
fmt.Println(u.Intro())
Analogi: struct itu kartu profil pemain, method itu aksi yang bisa dilakukan pemain tersebut.

BAB E. Pointers Methods Interfaces

E.1 Pointer Fundamentals

Pointer menyimpan alamat memori. Di Go pointer dipakai supaya perubahan data bisa langsung kena objek asli tanpa copy besar-besaran.

x := 10
p := &x

fmt.Println(x)
fmt.Println(*p)

*p = 20
fmt.Println(x)
Analogi: pointer itu kayak share lokasi, bukan kirim ulang seluruh isi rumah.

E.2 Pointer to Struct and Mutating Data

Untuk update field struct secara efisien, receiver pointer sering dipakai. Ini penting saat objek mulai besar atau sering diubah.

type Account struct {
    Name  string
    Point int
}

func (a *Account) AddPoint(v int) {
    a.Point += v
}

acc := Account{Name: "Nina", Point: 10}
acc.AddPoint(5)
fmt.Println(acc.Point)
Analogi: daripada fotokopi dokumen tiap revisi, mending edit langsung dokumen aslinya.

E.3 Interfaces and Implicit Implementation

Go tidak butuh keyword khusus untuk implement interface. Selama struct punya method yang diminta interface, dia otomatis kompatibel.

type Speaker interface {
    Speak() string
}

type Cat struct{}

func (Cat) Speak() string {
    return "meong"
}

func say(s Speaker) {
    fmt.Println(s.Speak())
}

say(Cat{})
Analogi: selama lo bisa memenuhi syarat masuk, lo boleh ikut pertandingan tanpa daftar label tambahan.

E.4 Polymorphism in Go Design

Polymorphism di Go muncul lewat interface. Ini bikin desain lebih fleksibel dan gampang diuji, karena fungsi cukup bergantung pada perilaku, bukan tipe konkret.

type Payment interface {
    Pay(amount int) string
}

type Cash struct{}

type EWallet struct{}

func (Cash) Pay(amount int) string {
    return fmt.Sprintf("Cash %d", amount)
}

func (EWallet) Pay(amount int) string {
    return fmt.Sprintf("EWallet %d", amount)
}
Analogi: kasir cuma peduli transaksi berhasil, mau bayar tunai atau dompet digital jalurnya tetap bisa diproses.

BAB F. Error Handling Packages and Files

F.1 Error Handling with if err != nil

Go tidak pakai exception sebagai alur utama. Pola umum adalah return error lalu langsung cek dengan if err != nil.

func divide(a int, b int) (int, error) {
    if b == 0 {
        return 0, fmt.Errorf("pembagi tidak boleh nol")
    }
    return a / b, nil
}

res, err := divide(10, 0)
if err != nil {
    fmt.Println("error:", err)
    return
}
fmt.Println(res)
Analogi: tiap langkah kerja ada checklist. Kalau ada masalah, berhenti dulu dan lapor jelas, jangan pura-pura aman.

F.2 Custom Errors and Wrapping Context

Custom error berguna saat domain butuh pesan yang lebih kaya. Error wrapping membantu jejak masalah tetap kebaca sampai level atas.

var ErrNotFound = errors.New("data tidak ditemukan")

func findUser(id int) error {
    return fmt.Errorf("find user %d: %w", id, ErrNotFound)
}
Analogi: bukan cuma bilang "barang hilang", tapi kasih konteks barang apa dan kejadian di mana.

F.3 Packages Modules and Standard Library

Package dipakai buat membagi kode berdasarkan tanggung jawab. Module lewat go.mod ngunci dependency supaya build konsisten lintas mesin.

go mod init contoh-app

go get github.com/google/uuid

go mod tidy
Analogi: package itu pembagian ruangan kerja, module itu buku inventaris resmi semua alat yang dipakai.

F.4 File Handling Basics

Go punya package os dan io yang kuat buat baca tulis file. Untuk operasi simple, os.ReadFile dan os.WriteFile sudah cukup.

content := []byte("belajar golang step by step")
err := os.WriteFile("catatan.txt", content, 0644)
if err != nil {
    fmt.Println(err)
    return
}

data, err := os.ReadFile("catatan.txt")
if err != nil {
    fmt.Println(err)
    return
}
fmt.Println(string(data))
Analogi: tulis file itu kayak simpan nota ke arsip, baca file itu ngambil lagi arsip pas dibutuhkan.

BAB G. Concurrency with Goroutines and Channels

G.1 Goroutines Fundamentals

Goroutine is a function that runs concurrently with low overhead. This is the main reason Go is strong in services that need many parallel processes.

func kerja(nama string) {
    fmt.Println("mulai", nama)
    time.Sleep(300 * time.Millisecond)
    fmt.Println("selesai", nama)
}

go kerja("A")
go kerja("B")
time.Sleep(1 * time.Second)
Analogi: satu dapur dengan banyak koki yang bisa masak menu berbeda di waktu bareng.

G.2 Channels for Safe Communication

Channel is used for communication between goroutines without data race conditions. This makes concurrent patterns safer and easier to understand.

ch := make(chan string)

go func() {
    ch <- "pesan dari goroutine"
}()

msg := <-ch
fmt.Println(msg)
Analogy: channel is like official package delivery route, so items arrive at destination without random throwing.

G.3 Buffered Channels and Select

Buffered channel can wait for data before being read. Select is used when you wait for many channels at once.

ch1 := make(chan string, 1)
ch2 := make(chan string, 1)

ch1 <- "dari ch1"
ch2 <- "dari ch2"

select {
case m1 := <-ch1:
    fmt.Println(m1)
case m2 := <-ch2:
    fmt.Println(m2)
default:
    fmt.Println("no messages yet")
}
Analogi: select itu seperti operator call center yang siap terima telepon dari beberapa jalur masuk.

G.4 Common Concurrency Pitfalls

Common bugs: deadlock, goroutine leak, and race condition. Always design clear channel flows and close channels when finished using.

go test -race ./...
  • Avoid sending to channels that are never read.
  • Use context to cancel long operations.
  • Audit data ownership accessed by many goroutines.
Analogy: if everyone talks randomly without rules, it will definitely be chaos.

BAB H. HTTP JSON and Testing

H.1 Building Simple HTTP Server

Package net/http is sufficient to build simple server without additional frameworks. Suitable for understanding web foundations in Go.

http.HandleFunc("/ping", func(w http.ResponseWriter, r *http.Request) {
    fmt.Fprintln(w, "pong")
})

http.ListenAndServe(":8080", nil)
Analogi: net/http itu dapur dasar, lo sudah bisa buka warung tanpa dekor mewah dulu.

H.2 Routing and JSON Response

API endpoints usually send JSON responses. Encoding and decoding JSON is a mandatory skill for Go backend.

type Resp struct {
    Message string `json:"message"`
}

http.HandleFunc("/hello", func(w http.ResponseWriter, r *http.Request) {
    w.Header().Set("Content-Type", "application/json")
    json.NewEncoder(w).Encode(Resp{Message: "halo"})
})
Analogy: JSON is a standard package format, so different systems can still understand the contents of sent messages.

H.3 JSON Decode with Struct Tags

Struct tags ensure Go fields can match with JSON keys that are usually snake_case or lower case.

type User struct {
    FullName string `json:"full_name"`
    Age      int    `json:"age"`
}

raw := []byte(`{"full_name":"Rani","age":24}`)
var u User
err := json.Unmarshal(raw, &u)
if err != nil {
    fmt.Println(err)
    return
}
fmt.Println(u.FullName, u.Age)
Analogi: struct tag itu label penerjemah biar format luar dan format internal bisa nyambung rapi.

H.4 Basic Unit Testing in Go

Testing in Go is simple and done directly through the testing package. Usually write small tests from the beginning to make refactoring safer.

// file: math_test.go
func TestTambah(t *testing.T) {
    got := tambah(2, 3)
    want := 5

    if got != want {
        t.Fatalf("got %d want %d", got, want)
    }
}

// jalankan:
// go test ./...
Analogy: testing is like quality check before shipping products to customers.

BAB I. Suggested Path and Practice

I.1 Suggested Learning Path

So learning doesn't jump around, here's the most logical order:

  1. Basics first: intro, syntax, variable, control flow, loop.
  2. Move to functions, slices, maps, structs, interfaces.
  3. Strengthen error handling and package modules.
  4. Learn concurrency gradually until you understand patterns.
  5. Finish with HTTP, JSON, and testing.
Analogy: don't jump straight to the main league, build stamina from consistent basic exercises.

I.2 Practice Project Ideas

Project kecil yang realistis dan efektif buat ngunci konsep:

  • CLI calculator dengan validasi input.
  • File processor untuk baca dan olah text log.
  • REST API CRUD sederhana.
  • Concurrent worker system untuk proses job batch.
# contoh jalur latihan
week1: basic + function
week2: slice map struct
week3: interface + error handling
week4: goroutine channel + mini API
Analogi: skill ngoding itu kebentuk dari repetisi drill, bukan dari nonton teori doang.

I.3 Practical Tips for Fast Improvement

  • Fokus dulu kuasai slice, interface, dan error handling.
  • Tulis kode sedikit tapi rutin tiap hari.
  • Pahami dulu standar library sebelum buru-buru framework.
  • Biasakan profiling dan test saat project mulai gede.

Kalau konsisten di fondasi ini, transisi ke production-grade service bakal jauh lebih mulus.

Analogi: pemain yang latihan dasar bener biasanya lebih stabil pas tanding dibanding yang cuma jago trik.
Kesimpulan: Kekuatan Go ada di kombinasi simplicity, performa, dan concurrency. Kuasai fondasi dulu, lalu naik bertahap ke API dan project concurrent. Dengan pola latihan terarah, skill Go lo bakal cepat matang dan siap dipakai di kerjaan nyata.
