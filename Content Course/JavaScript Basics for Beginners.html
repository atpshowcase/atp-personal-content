BAB A. Introduction and Setup

A.1 What is JavaScript

JavaScript adalah bahasa pemrograman utama untuk web interaktif. HTML ngatur struktur, CSS ngatur tampilan, dan JavaScript bikin halaman jadi hidup.

Sekarang JavaScript bukan cuma buat browser, tapi juga dipakai di backend lewat Node.js.

console.log("Halo JavaScript");
Analogi: kalau website itu cafe, HTML adalah bangunan, CSS dekorasi, JavaScript adalah staf yang gerak melayani pelanggan.

A.2 History Evolution and Engine

JavaScript berkembang cepat dari bahasa scripting sederhana jadi fondasi aplikasi modern. Di balik layar, kode JS dijalankan oleh engine seperti V8 di Chrome dan Node.js.

  • Bahasa berkembang lewat standar ECMAScript.
  • Fitur modern muncul bertahap lewat ES6 dan seterusnya.
  • Tooling makin matang untuk frontend dan backend.
// cek versi node
node -v
Analogi: engine itu mesin motor. Body bisa sama, tapi performa tergantung mesin yang jalanin.

A.3 Setup Environment

Setup belajar JavaScript paling simpel: browser + DevTools + editor. Buat backend, install Node.js agar bisa run JavaScript di terminal.

// jalanin file js
node app.js
Analogi: ini kayak nyiapin meja kerja, lampu, dan alat tulis sebelum mulai belajar serius.

BAB B. Syntax Variables and Operators

B.1 Basic Syntax and Structure

JavaScript terdiri dari statement dan expression. Bahasa ini case-sensitive, jadi nama variable huruf besar kecil harus konsisten.

// ini komentar satu baris
/*
  ini komentar multi baris
*/

let message = "Belajar JS";
console.log(message);
Analogi: beda huruf besar kecil itu kayak beda nama orang. Raka dan raka dianggap dua entitas berbeda.

B.2 var let const and Data Types

Gunakan let untuk variable yang bisa berubah, const untuk nilai yang tidak direassign, dan hindari var untuk kode modern.

let age = 24;
const appName = "BelajarJS";

let text = "halo";
let num = 10;
let isActive = true;
let empty = null;
let notSet;

console.log(typeof text, typeof num, typeof isActive);
Analogi: let itu sticky note yang bisa diganti, const itu label permanen di map dokumen.

B.3 Primitive and Reference Types

Primitive disalin berdasarkan nilai, sedangkan object dan array disalin berdasarkan referensi.

let a = 5;
let b = a;
b = 8;

const user1 = { name: "Ayu" };
const user2 = user1;
user2.name = "Bima";

console.log(a, b);
console.log(user1.name);
Analogi: primitive itu fotokopi kertas, reference itu dua orang pegang alamat rumah yang sama.

B.4 Operators and Comparison

Pahami perbedaan == dan ===. Untuk kode aman, prioritaskan === agar tipe juga ikut dibandingkan.

const x = 5;
const y = "5";

console.log(x == y);  // true
console.log(x === y); // false

const result = (x > 3) && true;
const label = result ? "valid" : "tidak valid";
console.log(label);
Analogi: == itu cek wajah sekilas, === itu cek wajah plus KTP.

BAB C. Control Flow Looping and Functions

C.1 if switch and Truthy Falsy

Control flow ngatur alur program berdasarkan kondisi. JavaScript juga punya konsep truthy dan falsy yang penting buat validasi.

const input = "";

if (!input) {
  console.log("Input kosong");
} else {
  console.log("Input ada");
}

const role = "admin";
switch (role) {
  case "admin":
    console.log("Akses penuh");
    break;
  default:
    console.log("Akses terbatas");
}
Analogi: seperti satpam yang buka jalur berbeda tergantung kartu akses yang ditunjukkan.

C.2 Looping Patterns

Pakai jenis loop sesuai konteks: for untuk hitungan, while untuk kondisi, for...of untuk iterable, dan for...in untuk key object.

for (let i = 0; i < 3; i++) {
  console.log("for", i);
}

const arr = ["a", "b", "c"];
for (const item of arr) {
  console.log("for of", item);
}

const obj = { a: 1, b: 2 };
for (const key in obj) {
  console.log(key, obj[key]);
}
Analogi: beda jalur pengantaran untuk paket satuan, paket rutin, dan inventaris gudang.

C.3 Function Declaration Expression Arrow

Function declaration cocok untuk definisi utama, function expression untuk fleksibilitas, arrow function ringkas untuk callback.

function add(a, b) {
  return a + b;
}

const multiply = function (a, b) {
  return a * b;
};

const subtract = (a, b) => a - b;

console.log(add(2, 3));
console.log(multiply(2, 3));
console.log(subtract(5, 2));
Analogi: tiga cara pesan makanan, hasilnya sama, tinggal pilih jalur yang paling pas konteksnya.

C.4 Parameters Defaults and Rest

Default parameter bikin fungsi aman saat argumen tidak dikirim. Rest parameter ngumpulin banyak argumen jadi satu array.

function greet(name = "Teman") {
  return `Halo ${name}`;
}

function sumAll(...nums) {
  return nums.reduce((acc, n) => acc + n, 0);
}

console.log(greet());
console.log(sumAll(1, 2, 3, 4));
Analogi: default itu menu paket standar, rest itu meja prasmanan yang bisa isi sebanyak kebutuhan.

BAB D. Arrays Objects DOM and Events

D.1 Arrays and Common Methods

Array adalah struktur data dasar yang wajib dikuasai. Method seperti map filter reduce sering banget dipakai di project nyata.

const numbers = [1, 2, 3, 4, 5];

const doubled = numbers.map((n) => n * 2);
const evens = numbers.filter((n) => n % 2 === 0);
const total = numbers.reduce((acc, n) => acc + n, 0);

console.log(doubled, evens, total);
Analogi: map itu proses semua pesanan, filter itu pilih pesanan tertentu, reduce itu hitung total omset.

D.2 Objects Properties Methods Destructuring

Object dipakai untuk data yang punya pasangan key value. Destructuring bikin akses data lebih ringkas.

const user = {
  name: "Rani",
  city: "Bandung",
  greet() {
    return `Halo, ${this.name}`;
  }
};

const { name, city } = user;
console.log(name, city);
console.log(user.greet());
Analogi: object itu profil anggota, setiap properti seperti kolom data di kartu member.

D.3 DOM Selection and Manipulation

DOM manipulation adalah jembatan JavaScript ke UI browser. Lo bisa pilih elemen, ubah isi, style, sampai bikin elemen baru.

const title = document.getElementById("title");
const btn = document.querySelector(".btn");

title.textContent = "Belajar DOM";
title.style.color = "teal";

const li = document.createElement("li");
li.textContent = "Item baru";
document.querySelector("ul").appendChild(li);
Analogi: DOM itu panggung acara, JavaScript jadi kru yang ganti dekor, lampu, dan susunan properti secara live.

D.4 Event Handling and Propagation

Event bikin halaman responsif terhadap aksi user. Pahami bubbling capturing dan preventDefault agar event tidak bikin efek samping aneh.

const form = document.querySelector("form");
const button = document.querySelector("button");

button.addEventListener("click", () => {
  console.log("Button clicked");
});

form.addEventListener("submit", (e) => {
  e.preventDefault();
  console.log("Form submit ditahan");
});
Analogi: event bubbling itu kabar dari meja bawah naik ke manajer, preventDefault itu izin menahan proses otomatis.

BAB E. Modern JavaScript ES6 Plus

E.1 Template Literals and Optional Chaining

Template literal bikin string interpolation lebih enak dibaca. Optional chaining bikin akses data nested lebih aman tanpa error panjang.

const name = "Dita";
const score = 90;

const message = `Halo ${name}, skor kamu ${score}`;
console.log(message);

const profile = { social: { instagram: "@dita" } };
console.log(profile?.social?.instagram);
console.log(profile?.social?.tiktok);
Analogi: optional chaining itu seperti jalan cadangan, kalau jalan utama putus lo tidak langsung jatuh.

E.2 Destructuring Spread and Rest

Destructuring memecah data jadi variable, spread menyalin atau gabung data, rest menangkap sisa data.

const arr = [1, 2, 3, 4];
const [first, second, ...others] = arr;

const user = { name: "Ayu", city: "Jogja" };
const clone = { ...user, role: "admin" };

console.log(first, second, others);
console.log(clone);
Analogi: seperti bongkar paket besar jadi bagian kecil, lalu rakit lagi sesuai kebutuhan.

E.3 Modules import export

Module bikin kode terorganisir per file sehingga maintainability jauh lebih baik saat project membesar.

// math.js
export function add(a, b) {
  return a + b;
}

// app.js
import { add } from "./math.js";
console.log(add(2, 3));
Analogi: module itu kayak lemari terpisah per kategori barang, jadi pas cari tidak acak-acakan.

BAB F. Async Error Handling and APIs

F.1 Callbacks Promises async await

Asynchronous JavaScript dipakai saat operasi butuh waktu seperti network request. Gaya modern paling nyaman: async await di atas Promise.

function wait(ms) {
  return new Promise((resolve) => {
    setTimeout(resolve, ms);
  });
}

async function run() {
  console.log("Mulai");
  await wait(500);
  console.log("Selesai tunggu");
}

run();
Analogi: lo pesan kopi, sambil nunggu tetap bisa ngobrol. Tidak harus bengong di kasir.

F.2 Fetch API and JSON Handling

Fetch dipakai untuk ambil data HTTP. Biasakan cek status response sebelum parsing JSON agar alur error jelas.

async function getUsers() {
  const res = await fetch("https://jsonplaceholder.typicode.com/users");

  if (!res.ok) {
    throw new Error("Gagal ambil data");
  }

  const data = await res.json();
  console.log(data);
}

getUsers();
Analogi: fetch itu seperti kurir ambil paket dari gudang API lalu menyerahkan isi paket dalam format JSON.

F.3 try catch throw and Debugging

Error handling yang rapi bikin aplikasi tidak gampang crash diam-diam. Gunakan try catch untuk menangkap error async dan throw untuk validasi bisnis.

function parseAge(value) {
  const age = Number(value);

  if (Number.isNaN(age)) {
    throw new Error("Umur harus angka");
  }

  return age;
}

try {
  const result = parseAge("abc");
  console.log(result);
} catch (err) {
  console.error(err.message);
}
Analogi: try catch itu seperti safety net saat atraksi. Kalau ada yang meleset, tetap ada penahan.

F.4 Working with APIs and Basic CRUD

Untuk CRUD sederhana, pahami method GET POST PUT DELETE. Ini modal utama saat bikin dashboard admin atau app data-driven.

// GET
fetch("/api/tasks");

// POST
fetch("/api/tasks", {
  method: "POST",
  headers: { "Content-Type": "application/json" },
  body: JSON.stringify({ title: "Belajar API" })
});
Analogi: API itu kasir restoran online. Lo kirim pesanan, kasir proses, dapur kirim hasil balik.

BAB G. Backend Intro and Practice Roadmap

G.1 Introduction to Node.js and Express

Node.js memungkinkan JavaScript jalan di server. Express sering dipakai untuk bikin API cepat dan minimalis.

const express = require("express");
const app = express();

app.get("/ping", (req, res) => {
  res.json({ message: "pong" });
});

app.listen(3000);
Analogi: kalau browser itu front office, Node.js itu dapur belakang yang nyiapin data pesanan.

G.2 Suggested Learning Path

  1. Kuasai basic syntax sampai control flow.
  2. Lanjut ke function arrays objects.
  3. Fokus DOM dan events untuk interaktif.
  4. Naik ke async dan API integration.
  5. Opsional: masuk backend Node.js.
// checklist mingguan
// week1: basic + variable + operator
// week2: function + array + object
// week3: DOM + events
// week4: async + fetch + mini project
Analogi: latihan skill itu seperti naik level game, jangan lompat boss stage sebelum kuasai kontrol dasar.

G.3 Practice Project Ideas

Project kecil lebih efektif daripada teori panjang. Pilih satu ide, selesaikan end-to-end, lalu ulang dengan fitur tambahan.

  • Calculator berbasis DOM.
  • To-Do List dengan event dan local storage.
  • Weather app pakai API publik.
  • Notes app dengan CRUD sederhana.
Analogi: skill ngoding kebentuk dari repetisi sparring, bukan cuma nonton tutorial.

BAB H. Tips and Closing

H.1 Practical Tips for Fast Progress

  • Latihan langsung di browser untuk lihat feedback cepat.
  • Fokus kuat di DOM dan async karena ini paling sering dipakai.
  • Biasakan baca error message pelan-pelan, jangan panik.
  • Bangun mini project rutin daripada sekali bikin project terlalu besar.
// pola belajar harian 45 menit
// 15 menit baca konsep
// 20 menit praktik kode
// 10 menit review error dan catatan
Analogi: progres coding itu seperti latihan gym. Konsisten 30-45 menit per hari lebih efektif daripada maraton sebulan sekali.

H.2 What to Bring Home

JavaScript fundamentals bukan soal hafalan sintaks, tapi kemampuan berpikir alur, ngolah data, dan bikin UI responsif yang nyambung ke API.

Kalau fondasi ini kuat, pindah ke framework seperti React atau backend Node.js jadi jauh lebih ringan.

Kesimpulan: kuasai dasar JavaScript secara bertahap, banyak praktik kecil, dan biasakan debugging. Dengan ritme ini, lo bakal siap naik ke level project real-world dengan percaya diri.
