HTTP Methods Explained: GET, POST, PUT & DELETE

Hey everyone, ATP here. Today, let's learn about four HTTP methods every beginner developer should know together.

If you're new to APIs, you've probably seen GET, POST, PUT, and DELETE and wondered...

"Do I really need to learn all of these?"

The answer is yes—but don't worry. They're actually much simpler than they sound.

1. GET

When your application needs to retrieve data from a server, you'll use GET.

Think of it like asking:

"Hey server, can I see the data?"

GET only reads information. It doesn't change anything on the server. 

GET /users

For example, when you open an online store to browse products, your app is making a GET request.

2. POST

Now let's say you want to create something new.

Maybe you're creating an account. Sending a message. Uploading a photo.

That's where POST comes in. 

POST /users

POST sends new data to the server so it can create a new resource.

3. PUT

Sometimes the data already exists. You just want to update it.

That's exactly what PUT is for. 

PUT /users/1

Imagine your profile says you're 20 years old. After your birthday, you update it to 21.

You're replacing the existing information—not creating another profile.

4. DELETE

Sometimes you don't need the data anymore.

Maybe you want to delete a post. Remove an account. Delete a product.

That's where DELETE comes in. 

DELETE /users/1

DELETE tells the server to remove an existing resource.

Just... maybe don't try this on your production database. Your teammates will definitely remember you.

Quick Summary

Method Purpose
GET Read data
POST Create new data
PUT Update existing data
DELETE Remove data
  • ✅ GET → Read
  • ✅ POST → Create
  • ✅ PUT → Update
  • ✅ DELETE → Remove

Closing

If you remember one thing from today, remember this: GET reads, POST creates, PUT updates, and DELETE removes.

These four methods make up the majority of API requests you'll work with as a software engineer.
