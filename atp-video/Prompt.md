## Style Reference

Create a compelling narrative-driven video script that tells a cinematic story, but heavily relies on the fast-paced, highly engaging public content style of Jason Cooperson's Instagram:

https://www.instagram.com/jasoncooperson/

This video should tell a narrative story (with a clear beginning, middle, and end, using characters, plot, or visual metaphors) rather than just being a direct-to-camera lecture. However, the pacing, rhythm, and hook structure should feel exactly like a top-tier short-form creator.

Draw inspiration from:
- Character-driven storytelling blended with fast-paced creator rhythm
- Strong curiosity hooks that grab attention in the first 3 seconds
- Fast-paced storytelling with short, punchy sentences
- One idea introduced at a time
- Story-first, explanation-second
- Real-world analogies that simplify technical concepts while driving the plot forward
- Curiosity gaps that keep viewers watching
- A satisfying "Aha!" moment near the end
- Cinematic visual descriptions (camera angles, fast cuts, lighting) paired with dynamic pacing

Do NOT copy or closely imitate:
- Specific wording or sentences from the creator
- Catchphrases or personal opinions
- Standard "Hey guys" direct-to-camera educational lectures
- Dry, textbook explanations

The finished script should feel like an original, fast-paced narrative short film that captures the high-retention qualities of Jason Cooperson's style while developing its own unique voice.

-----------------------------------

Formatting Rules

- Use a clear scene-by-scene script format (like A/V script formatting).
- Clearly separate [VISUAL/ACTION] and [AUDIO/DIALOGUE].
- Describe the setting, characters, and mood clearly, keeping in mind the fast cuts and energetic pacing.

Code blocks

If code must be shown on screen, it should be integrated into the narrative visually.
When writing code examples for the screen, format them like this:

<pre><code>

code here

</code></pre>

with rounded black backgrounds.

-----------------------------------

Output

Return ONLY a complete HTML document containing the formatted script.

Include all CSS inside the HTML.
Do not explain anything.
Do not use external CSS frameworks.
Use Inter from Google Fonts.

The HTML should present the script beautifully, using a sleek, readable layout suitable for a production script.
